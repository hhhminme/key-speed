# 게임 출시 체크리스트 검수

앱인토스 게임 미니앱 출시 가이드(https://developers-apps-in-toss.toss.im/checklist/app-game.md)를 기반으로 현재 프로젝트의 코드를 검수합니다.

## 검수 절차

아래 체크리스트 항목을 코드 기반으로 하나씩 확인하고, 각 항목에 대해 PASS / FAIL / N/A(해당 없음) / WARN(확인 필요) 상태를 판정해주세요.

---

## 1. 접속 및 사운드

- [ ] 미니앱이 10초 이내에 최초 화면까지 정상 로딩되는가
- [ ] 배경음(BGM), 효과음(SFX), 햅틱 등 필요 기능이 적용되어 있는가
- [ ] 무음 모드 또는 시스템 진동 비활성 상태에서도 의도한 동작이 유지되는가
- [ ] 사운드가 있을 경우, 사용자가 직접 On/Off 설정할 수 있는가
- [ ] 백그라운드 전환 시 사운드가 즉시 종료되는가
- [ ] 백그라운드에서 복귀 시 사운드가 정상 재생되는가

## 2. UX 및 내비게이션 바

- [ ] 우측 상단 닫기 버튼이 정상 노출되고 동작하는가
- [ ] Safe Area 영역을 침범하지 않는가 (iOS Dynamic Island 포함)
- [ ] 진입 시 바텀시트가 자동으로 노출되지 않는가
- [ ] 특정 화면 전환 시 바텀시트로 강제 사용자 행동을 유도하지 않는가
- [ ] 모든 화면에서 미니앱을 퇴출할 수 있는 경로가 있는가
- [ ] CTA 버튼을 눌렀을 때 어떤 행동이 이어지는지 예측 가능한가
- [ ] 자사 서비스 이동이나 자사 앱 설치 유도가 없는가
- [ ] 내비게이션 바 아이콘이 모노톤으로 통일되어 있는가

## 3. 게임 로그인

- [ ] 유저 식별자(getUserKeyForGame) 값을 확인하고 저장하여 게임이 정상 시작되는가
- [ ] 랭킹, 레벨, 스테이지 등 유저 플레이 기록이 유지되는가
- [ ] 종료 후 재접속 시 필요 데이터가 유지되는가

## 4. 서비스 이용 동작

- [ ] 인게임 화면이 풀스크린으로 구현되어 있는가
- [ ] 설정한 가로/세로 모드가 정상 작동하는가
- [ ] OS의 뒤로가기 제스처를 사용하지 않는가
- [ ] 불법성이나 선정성 콘텐츠가 없는가
- [ ] 모든 UI 컴포넌트가 정상 작동하고, 2초 이상 지연이 없는가
- [ ] React Native 앱의 경우, 사용 전 사용자 권한 동의를 취득하는가
- [ ] 권한 미동의 시에도 나머지 기능이 정상 작동하는가
- [ ] 비정상적인 네트워크/메모리 사용량 급증이 없는가
- [ ] 미니앱 종료 시 확인 모달이 노출되는가
- [ ] 점수 측정, 스테이지 클리어 등 주요 흐름에 오류가 없는가

## 5. 인앱 결제 (사용 시)

- [ ] 결제 진행 시 재생 중인 음악이 일시정지되는가
- [ ] 인앱 결제 주문 금액과 결제창에 표시되는 금액이 일치하는가
- [ ] 인앱 결제가 정상 진행되는가 (구글 테스트 환경 포함)
- [ ] 결제 완료 후 미니앱 복귀 시 결과가 정상 반영되는가
- [ ] 결제창 취소 시 주문 화면으로 정상 복귀하는가
- [ ] 잔액 부족 등 결제 실패 시 사유를 인지할 수 있는가
- [ ] 인앱 결제 취소가 정상 처리되는가
- [ ] 사용자가 결제 내역을 확인할 수 있는가
- [ ] 기기 변경 시에도 기존 결제 데이터가 유지되는가

## 6. 인앱 광고 (사용 시)

- [ ] 광고 재생 시 미니앱 음악이 일시정지되는가
- [ ] 사용자가 예상하기 어려운 순간에 광고가 노출되지 않는가
- [ ] 광고가 정상 노출되고, 중간에 종료되거나 튕기는 현상이 없는가
- [ ] 광고가 사전 로딩되며, 재생 시점에 실시간 로딩하지 않는가
- [ ] 광고 종료 시 미니앱 화면으로 정상 복귀하는가
- [ ] 광고 종료 후 미니앱 음악이 정상 재생되는가
- [ ] 리워드 광고 끝까지 시청 시 보상이 정상 지급되는가

## 7. 공유 리워드 (사용 시)

- [ ] 설정한 공유 리워드 화면이 정상 노출되는가
- [ ] 공유 리워드 화면 종료 시 미니앱으로 정상 복귀하는가
- [ ] 친구 초대 완료 시 보상이 정상 지급되는가

## 8. 웹보드 게임 (해당 시)

- [ ] 성인 인증 기능이 연동되어 있는가
- [ ] 사용자 문의 채널(고객센터)이 제공되는가
- [ ] 확률형 아이템의 확률 정보를 사용자가 확인할 수 있는가

---

## 검수 방법

1. 프로젝트의 소스 코드를 분석하여 각 항목의 구현 여부를 확인합니다.
2. `granite.config.ts` (또는 관련 설정 파일)의 설정을 검토합니다.
3. 사운드 관련 코드(Audio API, Howler.js 등)를 검색하여 사운드 제어 구현을 확인합니다.
4. UI 컴포넌트와 레이아웃에서 Safe Area, 풀스크린 등의 처리를 확인합니다.
5. `getUserKeyForGame` 등 게임 로그인 API 사용 여부를 확인합니다.
6. 인앱 결제/광고 관련 SDK 연동 코드를 확인합니다.

## 출력 형식

아래 형식으로 각 항목의 검수 결과를 정리해주세요:

```
## 검수 결과 요약

| 카테고리 | PASS | FAIL | WARN | N/A |
|---------|------|------|------|-----|
| 접속 및 사운드 | 0 | 0 | 0 | 0 |
| UX 및 내비게이션 바 | 0 | 0 | 0 | 0 |
| ... | ... | ... | ... | ... |

### 상세 결과

#### 1. 접속 및 사운드
- PASS: 항목 설명
- FAIL: 항목 설명 → 수정 필요 사항
- WARN: 항목 설명 → 코드에서 확인 불가, 수동 테스트 필요
- N/A: 항목 설명 → 해당 기능 미사용
```

마지막에 **FAIL 항목에 대한 수정 제안**을 코드 예시와 함께 제시해주세요.
